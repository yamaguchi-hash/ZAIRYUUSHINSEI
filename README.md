# 在留資格申請書類作成システム

出入国在留管理庁（入管）に提出する在留資格関連の申請書類を、Webブラウザ上で効率的・正確・セキュアに一元管理・作成・出力するフルスタックシステムです。

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| フロント＆バックエンド | Next.js 15 (App Router) + React 19 |
| データベース | Neon (PostgreSQL) |
| ORM | Drizzle ORM |
| 認証 | NextAuth.js v5 (JWT) |
| スタイリング | Tailwind CSS v4 |
| デプロイ | Vercel |

## セットアップ手順

### 1. 環境変数の設定

```bash
cp .env.example .env.local
```

`.env.local` を編集し、以下を設定してください：

```env
# Neon PostgreSQLのコネクション文字列
DATABASE_URL="postgresql://..."

# NextAuth.jsのシークレット（openssl rand -base64 32 で生成）
AUTH_SECRET="your-secret-key"
NEXTAUTH_URL="http://localhost:3000"
```

### 2. 依存関係のインストール

```bash
npm install
```

### 3. データベースのセットアップ

```bash
# スキーマをDBに適用
npm run db:push

# 初期データの投入
npx tsx scripts/seed.ts
```

### 4. 開発サーバーの起動

```bash
npm run dev
```

ブラウザで http://localhost:3000 を開く

## 11ステップワークフロー

1. **基本情報セットアップ** — 申請人・所属機関の検索/新規登録
2. **書類リスト自動生成** — ISA準拠マスターから推奨書類を自動生成
3. **専門家チェック（✓）** — 行政書士が必要書類を目視確認・確定
4. **書類アップロード** — 申請者・企業が書類をアップロード
5. **AI-OCR解析** — アップロード書類からデータを自動抽出
6. **自動ドラフト生成** — OCRデータを申請書フォームに流し込み
7. **動的質問書作成** — 不足情報を多言語で申請者に問い合わせ
8. **回答の自動統合** — 質問書の回答を申請書に自動補完
9. **整合性チェック** — パスポート番号・氏名等の自動照合
10. **プレビュー・最終承認** — 不一致を赤色アラートで表示、専門家が承認
11. **提出パッケージ出力** — 入管指定順序でPDFを自動編成

## マルチロール権限

| ロール | 主な権限 |
|--------|---------|
| 外国人申請者 | マイページアクセス、書類アップロード、質問書回答 |
| 受入機関担当者 (HR) | 企業情報の入力・更新、進捗モニタリング |
| 専門家（行政書士等） | 書類チェック、レビュー・修正、最終承認、PDF出力 |
| システム管理者 | マスターデータのメンテナンス、ユーザー管理 |

## データモデル

- **OrganizationMaster** — 企業の最新基本情報
- **ApplicantMaster** — 外国人の最新基本情報
- **Application** — 申請案件（ステータス管理）
- **ApplicationSnapshot** — 承認時点のデータを固着保存（JSONB）
- **DocumentRequirementMaster** — ISA準拠の必要書類マスター
- **ApplicationDocumentChecklist** — 申請ごとの書類管理
- **AuditLog** — 人間による変更履歴の完全記録
- **PdfFieldMapping** — PDF座標マッピング管理

## Vercelへのデプロイ

```bash
vercel --prod
```

環境変数をVercelダッシュボードにも設定してください。

## セキュリティ

- **マルチテナントデータ分離**: 全クエリにtenantIDフィルター強制
- **通信暗号化**: SSL/TLS常時暗号化（Vercel標準）
- **個人情報保護**: Neon DB内での暗号化保存
- **セッション管理**: JWT + 8時間自動ログアウト
- **監査ログ**: 全変更操作を永続記録
